## 1. 최종 기획 요약

### 서비스 한 줄 설명

**RallyRoom은 팬이 직접 작은 응원방을 만들고, 투표·미션·응원 메시지에 참여해 포인트와 아이콘을 얻고, 마감 후 결과 카드를 남기는 팬 주도 커뮤니티 플랫폼이다.**

### 제출용 설명

초기에는 팬 커뮤니티 플랫폼을 넓게 검토했지만, 실존 스타나 작품을 무단으로 내세우는 외부 팬 투표 서비스는 공식성 문제와 보상 루프의 한계가 있다고 판단했다. 그래서 팬덤 전체를 비교하는 구조가 아니라, 팬이 직접 작은 응원방을 만들고 그 안에서 투표, 미션, 응원 메시지, 결과 카드를 운영하는 구조로 좁혔다.

RallyRoom은 팬이 만든 작은 응원 단위를 중심으로 포인트와 아이콘, 결과 카드, 응원 기록을 제공한다. 과제 MVP에서는 모든 대상과 데이터가 가상이며, 실제 스타나 작품과의 공식 관계를 주장하지 않는다. 장기적으로는 공식 계정과 크리에이터가 직접 응원방을 열고, 팬 참여 데이터를 뉴스형 콘텐츠, 캠페인 리포트, 팬덤 운영 인사이트로 확장할 수 있다.

## 2. 기획 배경과 판단 과정

### 2.1 검토했던 방향

초기 아이디어는 다음과 같았다.

- 팬덤 응원 캠페인 플랫폼
- 스트리머 팬서포트 보드
- 웹소설/IP 캐릭터 투표 및 AI 챗봇 해금 서비스
- 팬덤 응원 작전실 RallyRoom
- 팬 참여형 콘텐츠 구독 서비스
- 오픈형 팬 투표/토너먼트 플랫폼

### 2.2 기각 또는 보류한 이유

#### 광범위한 팬덤 플랫폼

아이돌, 스트리머, 웹소설 캐릭터, 스포츠팀, 게임 캐릭터를 한 플랫폼에서 같은 기준으로 비교하면 정당성이 약해진다. 응원 대상, 팬 행동, 보상 구조가 서로 다르기 때문이다.

#### 실존 스타 대상 투표 사이트

제휴 없이 특정 스타의 이름이나 이미지를 전면에 내세우면 공식성 오인, 초상권·상표권·퍼블리시티권 리스크가 생길 수 있다. 또한 팬이 투표해도 실제 스타에게 환원된다고 말하기 어렵다.

#### 스트리머 팬서포트 보드

실존 응원 대상이 있고 팬 행동이 스트리머 성장으로 연결될 수 있다는 점은 강하지만, 특정 플랫폼 의존성이 커진다. 또한 실제 스트리머와 제휴하지 않으면 “방송 밖 팬 활동 운영툴”이라는 전제가 약해진다. 장기적으로 공식 크리에이터 응원방 모델로 흡수하는 것이 적절하다.

#### CharaPick / 웹소설 캐릭터 IP 서비스

내부 IP 플랫폼에서 팬 투표 결과가 AI 챗봇, 외전, 일러스트 해금으로 이어지는 구조는 비즈니스 루프가 강하다. 다만 팬 주도 커뮤니티라는 과제 방향보다 “플랫폼이 후보를 내고 팬이 고르는 콘텐츠 구독형 서비스”에 가까워질 수 있다. 따라서 최종안으로 독립시키기보다 RallyRoom 안의 캐릭터/IP 특화 템플릿으로 흡수한다.

### 2.3 최종 판단

최종 중심축은 **RallyRoom 2.0**이다.

RallyRoom은 “팬덤 전체 순위”가 아니라 **응원방이라는 작은 단위**를 중심으로 작동한다. 팬은 큐레이션된 대상 DB에서 응원 대상을 고르고, 방 안에서만 투표·미션·메시지를 운영한다. 결과는 사이트 내부 포인트, 아이콘, 칭호, 결과 카드로 완결된다.

## 3. 서비스 포지셔닝

### 3.1 핵심 정의

**팬 주도 마이크로 응원방 플랫폼**

### 3.2 핵심 가치

팬 입장:

- 내가 좋아하는 대상에 대한 작은 응원방을 직접 만들 수 있다.
- 투표, 토너먼트, 티어리스트, 미션으로 재미있게 참여할 수 있다.
- 참여 기록이 포인트, 아이콘, 결과 카드로 남는다.
- 공식 투표가 아니어도 팬들끼리 작은 주제로 모일 수 있다.

방장/크루 입장:

- 응원방을 만들고 참여 데이터를 볼 수 있다.
- 팬 메시지, 후보 제안, 미션 완료율을 확인할 수 있다.
- 결과 카드를 통해 다음 응원방으로 이어지는 루프를 만들 수 있다.

플랫폼/비즈니스 입장:

- 팬 행동 데이터를 응원방 단위로 수집한다.
- 잘 작동하는 템플릿을 상품화할 수 있다.
- 공식 계정, 크리에이터, IP 운영자용 캠페인 도구로 확장할 수 있다.
- 팬 키워드 분석, 결과 카드, 뉴스형 요약, 운영자 인사이트로 확장할 수 있다.

## 5. 핵심 객체: 응원방

### 5.1 응원방 정의

**응원방은 하나의 응원 대상을 중심으로 팬들이 투표, 티어 평가, 토너먼트, 미션, 메시지에 참여하고, 마감 후 결과 카드와 개인 보상을 남기는 마이크로 팬 캠페인이다.**

### 5.2 응원방 구성요소

```text
응원방 = 응원 대상 + 진행 방식 + 투표 주제 + 후보 + 기간 + 미션 + 보상 + 결과 카드
```

### 5.3 응원 대상과 후보의 분리

응원 대상은 방의 중심이다.  
투표 후보는 그 응원방 안에서 선택되는 항목이다.

예시:

```text
응원 대상: 〈던전 옆 작은 식당〉
투표 주제: 이번 주 가장 응원하고 싶은 캐릭터는?
투표 후보: 미나, 카일, 토토, 라비, 노아, 엘리, 로웬, 세라
```

```text
응원 대상: 모카문
투표 주제: 다음 방송에서 보고 싶은 콘텐츠는?
투표 후보: 마인크래프트 건축, 신작 인디게임, 심야 라디오, 시청자 참여
```

### 5.4 핵심 정책

- 하나의 응원방은 하나의 주 응원 대상을 가진다.
- 투표 후보는 같은 상위 대상 또는 같은 주제 안에서만 구성한다.
- MVP에서는 큐레이션된 가상 대상 DB만 사용한다.
- 실존 스타, 실제 작품, 실제 브랜드를 직접 사용하지 않는다.
- 공식 전달, 공식 반영을 약속하지 않는다.
- 사이트 전체 통합 인기 랭킹을 만들지 않는다.
- 결과는 응원방 내부 기록과 결과 카드로 완결한다.

## 7. 진행 방식 설계

### 7.1 픽 투표

가장 기본적인 방식이다.

예시:

- 다음 방송에서 보고 싶은 콘텐츠는?
- 이번 주 가장 응원하고 싶은 캐릭터는?
- 외전으로 보고 싶은 소재는?

선택 방식:

- 단일 선택
- 멀티픽, 최대 3개

### 7.2 올려/내려 티어리스트

후보를 한 장씩 보고 “올려요 / 내려요 / 보류”로 평가한다.

예시:

- 입문자에게 추천할 장면인가요?
- 이 외전 소재를 보고 싶나요?
- 이 응원 문구를 대표 문구 후보로 올릴까요?

결과:

- S티어: 강력 추천
- A티어: 팬들이 좋아함
- B티어: 호불호 있음
- C티어: 다음 기회

### 7.3 랜덤 토너먼트

이상형 월드컵 방식이다.

예시:

- 작품 캐릭터 월드컵
- 이번 주 명장면 월드컵
- 신규 스킨 콘셉트 월드컵

정책:

- 후보 8개 이상일 때 권장
- 개인별 토너먼트 결과와 전체 우승 횟수를 분리한다.

### 7.4 응원 미션

투표와 별개로 반복 방문을 유도한다.

예시:

- 오늘 출석하기
- 투표 참여하기
- 응원 한 줄 남기기
- 질문 후보 남기기
- 결과 카드 확인하기

### 7.5 메시지/질문 수집

응원방의 팬월 역할이다.

예시:

- 캐릭터에게 묻고 싶은 질문
- 대표 응원 문구
- 입문 추천 한 줄 문구
- 팬 응원 메시지

## 8. 포인트와 보상 정책

### 8.1 Rally Point, RP

사용자 개인의 누적 참여 점수다.

용도:

- 프로필 레벨
- 칭호 해금
- 결과 카드 테마 해금
- 프로필 아이콘/프레임 해금
- 마이페이지 기록 강조

금지:

- 투표권 구매
- 후보 순위 직접 상승
- 유료 결제로 직접 충전

획득 예시:

| 행동 | RP |
|---|---:|
| 응원방 첫 참여 | +5 |
| 픽 투표 참여 | +10 |
| 티어 평가 완료 | +15 |
| 토너먼트 완주 | +30 |
| 오늘의 미션 완료 | +15 |
| 응원 메시지 작성 | +10 |
| 후보 제안 승인 | +25 |
| 결과 카드 확인 | +5 |
| 만든 방의 결과 카드 생성 | +50 |

정책:

```text
일일 기본 RP 획득 상한: 300 RP
방 하나당 일일 RP 상한: 100 RP
동일 유형 반복 보상: 1일 1회
```

### 8.2 Room Energy

응원방 공동 목표 게이지다.

획득 예시:

| 행동 | Room Energy |
|---|---:|
| 투표 참여 | +2 |
| 미션 완료 | +5 |
| 응원 메시지 작성 | +3 |
| 토너먼트 완주 | +8 |
| 후보 제안 승인 | +5 |

용도:

- 결과 카드 스페셜 테마 해금
- 방 히스토리 아카이브 등록
- 인기 메시지 TOP 3 노출
- 공동 목표 달성 배지

### 8.3 Badge / Icon / Title

내부 명예 보상이다.

예시:

- 최초 참여 배지
- 7일 연속 참여 배지
- 방 개척자 배지
- 토너먼트 완주자 배지
- 결과 카드 수집가 배지
- 응원 문구 채택자 배지

### 프롬프트 1: 전체 앱 1차 생성

```text
팬들이 직접 작은 응원방을 만들고, 투표와 미션으로 포인트와 아이콘을 모아 결과 카드를 남기는 팬 주도 커뮤니티 웹 서비스 "RallyRoom"을 만들어줘.

이 서비스는 실존 스타나 실제 작품을 사용하지 않는 과제용 데모이며, 모든 대상은 가상 데이터다. 공식 팬클럽이나 공식 투표처럼 보이면 안 된다. 핵심은 팬이 직접 주제를 만들고, 카테고리와 대상 DB에서 응원 대상을 선택해 작은 응원방을 운영하는 것이다.

서비스의 핵심 객체는 "응원방"이다.
응원방은 응원 대상, 진행 방식, 투표 주제, 후보, 기간, 미션, 보상, 결과 카드로 구성된다.

필요 화면:
1. 홈 대시보드
2. 응원방 만들기
3. 응원방 상세
4. 투표 및 미션 영역
5. 결과 카드
6. 마이페이지
7. 운영자/Crew 대시보드
8. 요금제 화면

디자인 톤:
- 팬덤스럽지만 과하게 공식 아이돌 앱처럼 보이지 않게 한다.
- 서비스형 SaaS의 신뢰감과 팬 커뮤니티의 즐거움을 섞는다.
- 첫 화면에서 "응원방 만들기"가 가장 중요한 행동으로 보여야 한다.
- 모바일에서도 보기 좋은 반응형 UI로 구성한다.
```

### 프롬프트 2: 응원방 만들기 구체화

```text
RallyRoom의 "응원방 만들기" 화면을 5단계 폼으로 구체화해줘.

Step 1. 무엇을 응원할까요?
- 카테고리 선택: 인물/크리에이터, 작품/IP, 캐릭터, 팀/이벤트, 커뮤니티 테마
```

## 18. 최종 판단

RallyRoom은 단순 팬 투표 사이트가 아니라 다음 구조를 가진다.

```text
팬 주도 응원방 생성
→ 투표/티어/토너먼트/미션 참여
→ 포인트와 아이콘 보상
→ Room Energy 공동 목표 달성
→ 결과 카드와 응원 기록 생성
→ 마이페이지와 Crew 대시보드에 데이터 축적
→ 공식 계정/크리에이터 캠페인으로 확장
```

따라서 과제 평가자가 볼 수 있는 포인트는 다음이다.

- 문제를 넓게 잡았다가 팬덤군 혼재 문제를 인식하고 좁혔다.
- 실존 대상 제휴 부재와 공식성 리스크를 고려했다.
- 투표 결과 환원 문제를 사이트 내부 보상과 결과 카드로 해결했다.
- 팬 주도성과 운영자 대시보드라는 양면 구조를 만들었다.
- MVP 구현 범위와 장기 비즈니스 확장을 분리했다.
